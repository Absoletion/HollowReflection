# v0.48 Remediation Contract

Source: `Hollow_Reflection_Implementation_Continuation_Playbook.md` (2026-07-18).

## Locked decisions

- Live real-time combat is canonical; round-based code is legacy-only.
- `STORY_REGISTRY.missions` plus durable `missionClears` own progression.
- Combat parties contain at most **four** units. This supersedes the playbook's five-unit recommendation.
- Hale begins at 4-star and evolves to 5-star UnHollowed in Chapter 6.
- Cinnia's 5-star Ascension must provide documented combat scaling; a unique kit remains optional.
- Reward-bearing changes commit durably before the UI applies or celebrates them.
- Permanent rewards use permanent domain markers, never bounded history as authority.
- Summoning unlocks after `act1_9` and is guarded at navigation, route, and command boundaries.
- `hollowing-demo.html` remains generated and untracked; source changes build the standalone.

## Integration order

1. State/save integrity (P0).
2. Fast UI/runtime defects and regression tests.
3. Canonical story authority.
4. Combat progression.
5. Content/readability.
6. Release hardening and documentation reconciliation.

No new economy or story content lands before the P0 state/progression defects pass the aggregate verification suite.

## Post-audit implementation status

Schema 7 is now the runtime save contract. `settledBattles`, `completedOperations`, and `storyRecruitments` are permanent authority maps; `completedTransactions` remains bounded recent history for diagnostics only. `GameState.execute()` validates raw input before normalization and validates the draft again before returning a result. Reward-bearing UI flows commit the command result before mutating or celebrating state.

Canonical story gates, fixed authored parties, finite side-mission rewards, locked-hub fallback, and Summon command enforcement are implemented. Browser smoke covers fresh/seeded navigation, the one-unit 1-1 launch, side-mission unlock and persistence, and a deliberately failed save commit. CI runs on pull requests plus `main`, `integration/v0.48-remediation`, and `fix/v0.48-post-audit` pushes.
